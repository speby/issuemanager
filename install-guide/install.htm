UltraApps Issue Manager v2.0 Beta Installation Guide (ASP.NET version)

System Requirements
1) Windows 2000 (Service Pack 2) / Windows XP
2) IIS
3) MDAC 2.7
4) .NET Framework SDK Beta2 / ASP.NET Premium Edition Beta2
5) WinZip or similar program

Installation

1) Download the IssueManager Zip file from http://www.ultraapps.com/appdownload1.asp?app_id=1. Once you agree to the license agreement you can download ASP.NET version of IM. 

2) Once you have the downloaded the zip file, extract the files to your hard disk, be sure you have 'Use folder names' option checked while unzipping. You can directly unzip the files into your IIS directory i.e. C:\InetPub\wwwroot\. This will automatically create a directory called IssueManager and extract all the application files within that directory. (If you are installing the application into some directory other than c:\Inetpub\wwwroot\IssueManager be sure to complete step number 5.)


Figure 1: Unzipping the files

3) Next, we convert the IssueManager directory into a Virtual Directory. On Windows 2000 you can use the Internet Services Manager available from Start -> Control Panel -> Administrative Tools -> Internet Services Manager. From within Internet Services manager select the Default Web Site tab and then from the Action menu choose New -> Virtual Directory.


Figure 2: Create a new Virtual Directory

4) In the New Virtual Directory Creation dialog click next and you will be presented with a screen as shown above in Figure 2. Enter the Alias as IssueManager and click next. Then enter the path to the actual directory containing the Web Application i.e. C:\Inetpub\wwwroot\IssueManager (in our example) and click next. Keep the Access Permissions at their default values and click next twice to finish the Virtual Directory creation.

5) In the Internet Services Manager panel select the IssueManager Virtual Directory you created in the above step and expand it to view all the files and folders. Now select the Uploads directory and right-click it and select Properties from the Context Menu. From the Directory Properties dialog select write to give this directory access to write files. You will need this so that you can upload files to the server.

 
Figure 3: Add Write Access

6) If you have installed the application into some directory other than C:\Inetpub\wwwroot\IssueManager then you will have to modify the path to the database file before your application installation is complete (others can skip this step). Start notepad (or any other text editor) and open the web.config file from the IssueManager directory.
Also if you want to use Ms SQL as your database you will have to update the below line to something like
Provider=SQLOLEDB.1;Persist Security Info=False;User ID=sa;Initial Catalog=IssueManager;Data Source=Localhost
To use the MS SQL OleDb provider to connect to the IssueManager database.


Figure 4: web.config file

Now find the line highlighted in Figure 3 and replace it with the path to the database file in your installation. Once done, save and close the file.

7) The above steps complete the installation. To test your installation start your browser and enter the url to the IssueManager i.e http://localhost/IssueManager. If your installation went correct then you should be presented with a log-in screen similar to the one shown in Figure 4.


Figure 5: Login Screen

By default IssueManager has one administrator account with the username admin and password admin. Use these credentials to log-in for the first time.

7) Once you have logged-in go to the Administration panel and select Users. The next screen contains a list of available users. You can either edit a user profile by selecting a user or add a new user/moderator/administrator.


Figure 6: User List

8) Select the Administrator from the User List page, and from the Administrator Details page enter a new password in the password field and select Update to change the Administrator password. (We strongly recommend you change the Administrator Password in order to prevent misuse of your application!) Once you have added the necessary users your setup is complete and you can begin to use Issue Manager!


Figure 6: User Maintenance
